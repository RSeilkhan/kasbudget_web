Политика конфиденциальности KasBudget

KasBudget уважает право пользователей на конфиденциальность.

1. Локальная обработка пользовательских данных

Следующие данные обрабатываются только локально на устройстве пользователя и не передаются на внешние серверы:

  • загруженные пользователем PDF-выписки;
  • извлеченные из выписок транзакции (например: дата, сумма, мерчант, категория);
  • пользовательские настройки приложения, фильтры, профили и локальные отчеты.

Эти данные остаются только в локальной памяти/хранилище устройства пользователя.

2. Данные, собираемые через Firebase

Для работы и улучшения приложения используются сервисы Firebase (например, Analytics, Crashlytics, Remote Config).

Через Firebase могут собираться технические и служебные данные, включая:

  • данные о сбоях и ошибках приложения (crash logs, stack traces);
  • технические данные устройства и приложения (модель устройства, ОС, версия приложения, идентификатор экземпляра приложения);
  • данные взаимодействия с приложением (события использования);
  • служебные параметры для удаленной конфигурации (например, язык, страна/регион, версия платформы/приложения).

3. Передача третьим лицам

Пользовательские финансовые данные из выписок не передаются третьим лицам.

Технические данные, указанные в разделе 2, обрабатываются Firebase как поставщиком инфраструктуры.

4. Международная передача данных

Данные, обрабатываемые через Firebase, могут обрабатываться на серверах за пределами страны пользователя в соответствии с политиками Google/Firebase.

Privacy Policy — KasBudget

KasBudget respects users’ privacy.

1. Local Processing of User Data

The following data is processed locally on the user’s device only and is not sent to external servers:

  • PDF statements imported by the user;
  • transaction data parsed from statements (e.g., date, amount, merchant, category);
  • app settings, filters, profiles, and local reports.

This data remains only in the device’s local memory/storage.

2. Data Collected via Firebase

KasBudget uses Firebase services (such as Analytics, Crashlytics, and Remote Config) for app operation and improvement.

Firebase may collect technical/service data, including:

  • app crash and error data (crash logs, stack traces);
  • device and app technical data (device model, OS, app version, app instance identifier);
  • app interaction/usage events;
  • remote configuration related parameters (e.g., language, country/region, platform/app version).

3. Sharing with Third Parties

Financial user data from statements is not shared with third parties.

Technical data listed in section 2 is processed by Firebase as an infrastructure provider.

4. International Data Transfers

Data processed via Firebase may be processed on servers outside the user’s country according to Google/Firebase policies.
